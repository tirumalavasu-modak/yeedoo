Frequently Asked Questions

Is this suitable for large enterprises / data-heavy workloads?​

Yes, the challenge is specifically built for organizations with large-scale Spark usage who want to audit and reduce their cloud compute costs.​

Are there minimum thresholds for cost savings?

No, there are no minimum thresholds. Yeedu delivers noticeable cost savings when a mix of low, medium, and heavy workloads are run on the platform.​

What do you need from us to run the 30 Days Cost Savings Challenge?

We need three things to get started. Cloud infrastructure to install Yeedu. The right access permissions such as opening required firewall paths and read access to data sources. A set of representative Spark jobs so we can evaluate performance and cost savings accurately.​

Do I need to change my code or migrate to Yeedu?

No. Yeedu runs your existing Spark jobs as-is in your own cloud environment.​

Which clouds are supported?​

AWS, Azure, and GCP. Yeedu supports x86, ARM, and GPU runtimes for multi-cloud optimization.​

How is data security handled during the pilot?

The workloads run in your own cloud, you retain data control. Yeedu complies with ISO27001, HIPAA, SOC2, GDPR.​

What if I’m not satisfied with the results?

The no-cost pilot program offers minimal risk. If the realized savings do not meet your expectations, you are free to simply discontinue the program or discuss a more suitable plan with our team.​

How soon can I get started?

Once you sign up, Yeedu’s engineering team will schedule a 1:1 review and kick off the pilot right away.​
